# /dev-loop - 自主循环开发（一轮一最小提交）

配合内置 `/loop` 使用：**设计好需求文档 + 原型后**，一句话起飞，自动跑到完成。
本命令是「一轮的标准作业」，由 `/loop` 负责自动续期——干净复用原生循环，不自造。

> 本文件在母框架 `tauri` 内为**规范源**，经 `/sync-from-framework` 同步到各派生项目；改这里。

## 用法

- `/loop /dev-loop` —— 自主连续开发：每轮做一个最小可提交单元，自动续期，直到台账全勾完才停。
- `/dev-loop` —— 只手动跑一轮（调试 / 单步推进用）。

## 真相源（每轮开工必读，不可凭记忆、不可偏离）

1. `CLAUDE.md` + `.claude/PROJECT.md`（若存在）—— 技术栈与禁令
2. **需求文档** —— `docs/requirements/PRD-*.md`（取存在者）
3. **原型** —— `prototype/*.html`（照其末尾「组件映射表」注释实现，若有）
4. **任务台账** —— `docs/tasks/ROADMAP.md` 或 `docs/tasks/active/BUILD-PLAN.md` 或 `docs/tasks/active/*.md`（取存在者，`[ ]/[x]` 进度）

## 前置门（缺则不进循环）

- **需求 / 原型 / 台账都找不到** → 不要瞎猜瞎编。停下提示用户：「先设计好需求文档 + 原型，再 `/loop /dev-loop`」。
- **有需求 / 原型但无台账** → 走「第 0 轮自举」：读需求 + 原型，拆成**有序最小任务清单**（每条含：序号、阶段、一句话目标、依赖、涉及文件、对应 PRD 章节 / 原型文件、验收标准、`[ ]`），写入 `docs/tasks/active/BUILD-PLAN.md`，提交（`docs(plan): 初始化任务台账`）后**结束本轮**。

## 每轮标准流程（台账已存在时）

1. 读真相源，选台账里「最靠前、依赖已完成」的第一个未勾 `[ ]` 任务。**严禁一轮做多个功能。**
2. 读该任务对应的 PRD 章节 **+ 对应原型 `prototype/*.html`（逐区块对照，见下方「🎨 原型保真」）**，明确做成什么样、调哪些 Command。
3. 实现这一个最小单元（细则照 `/check` 与 `.claude/skills/`）：
   - **后端**：三层架构 commands→services→database，在 `lib.rs` 的 `generate_handler![]` 注册；错误用 `AppError`，**禁止 unwrap/panic**。
   - **前端**：types→lib/api→store→pages，UI 用 Ant Design 5 + Tailwind，invoke 封装在 `lib/api`、路径用 `@/`、禁止 `any`。
4. **验证门（必须全绿才提交）**：
   - 后端改动：`cargo check --manifest-path src-tauri/Cargo.toml`
   - 前端改动：`pnpm exec tsc --noEmit`（无 pnpm 用 `npx tsc --noEmit`）
   - **UI 改动：额外过『🎨 原型保真验证门』（见下方专节），不达标 = 未完成、不许提交。**
   - 红了当场修到绿。**每完成 5 个任务或每到里程碑边界，额外跑一次 `/check` 做规范回归。**
5. 更新台账：该任务 `[ ]`→`[x]`，补一行「实际改了什么 / 遇到的坑」，更新「当前进度」「下一步」。
6. **最小提交**：逐个 `git add` 本轮改动的**具体文件**（禁止 `git add -A` / `git add .`），`git commit`（Conventional Commits 带 scope，一句话说清这步改了什么）。
7. 输出本轮小结：✅完成哪个任务 | 📊进度 X/总 | ⏭️下一个。

## 🎨 原型保真（涉及 UI 的任务强制 — 最容易翻车，列为最高优先级之一）

> 实践教训：实现最容易「偏离原型」。本节是硬约束，不是建议。**原型是 UI 的最终标准。**

1. **逐元素对照，不凭印象**：动手前先打开对应 `prototype/*.html`，按其 **布局结构 / 配色 / 间距 / 字号字重 / 圆角 / 组件类型 / 文案 / 交互态** 逐项还原；照 html 末尾「组件映射表」注释选对应 AntD 组件。
2. **用令牌，不手填**：颜色 / 间距 / 字号一律用原型的设计令牌（CSS 变量 `var(--xxx)` / Tailwind 配置），**禁止自己编颜色值或像素间距**；原型缺的令牌先补进设计系统再用。
3. **禁止自由发挥**：不许「我觉得这样更好看 / 更合理」擅自改布局、改配色、增删元素、换组件类型。与原型冲突一律**以原型为准**；确有更优方案 → 记 `docs/BLOCKERS.md` 等人工确认，本轮仍按原型实现。
4. **原型保真验证门（UI 任务必过）**：实现后逐条比对该页面与原型 —— ① 整体布局 / 分区一致 ② 主色 / 辅色 / 状态色一致 ③ 间距 / 留白一致 ④ 字号 / 字重 / 行高一致 ⑤ 组件类型与原型一致（卡片是卡片、表格是表格）⑥ 文案与原型一致 ⑦ hover/active/loading/empty 等交互态覆盖。**任一不符 = 未完成，返工到达标再提交。**
5. **能截图就截图比对**：若可起前端（vite dev），截运行页面与原型 **并排比对**，比「自我感觉对齐了」可靠得多。

## 决策规则（遇模糊 / 缺信息按风险分流）

- **可逆 / 低风险**（命名、文件位置、等价实现）→ 按合理默认继续，记一行到 `docs/BLOCKERS.md`，不空等、不反复问。
- **不可逆 / 合规 / 架构级 / 缺外部凭据**（token、支付密钥）→ 停止循环，列出需要用户做什么。有免密替代就先跑通、把缺口标 TODO 跳过。

## 红线（绝不触碰）

- 不发明 PRD / 原型里没有的功能；技术栈不换（Tauri2 + Rust + React19 + AntD5 + Tailwind + Zustand + rusqlite）。
- 文件 UTF-8 无 BOM、中文注释可读。
- 不按名杀宿主进程（按端口 / PID 精准）；不 `git add -A`；不 `git stash` / `reset --hard` / `clean` 丢改动；本地提交不推远端（除非用户明确要求）。
- 严守项目 `.claude/PROJECT.md` 的领域禁令（**优先级最高**）。

## 卡点熔断

连续 **3** 轮无实质进展，或同一任务连续 **2** 轮验证门修不绿 → 停止循环，把错误原文 + 你的判断 + 建议方案报给用户。

## 停止条件

台账全部 `[x]` + 验收清单全绿 → 输出「✅ 全部计划功能已实现」+ 验收自检，并**停止循环（不再续期）**。
