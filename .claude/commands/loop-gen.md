# /loop-gen - 按你的当次意图，现生成一条量身定做的 /loop 提示词

你不一定每次都是「按需求文档建功能」——有时是**还原原型**、有时是**修一批 bug / 补测试 / 重构 / 迁移 / 清理**。
本命令听你一句意图，产出一条**填好真相源 / 每轮动作 / 验证门 / 停止条件**的 `/loop` 提示词，你直接粘贴起飞。

> 和邻居的分工：`/dev-loop` = 固定 SOP（按任务台账建功能，最常用）；**`/loop-gen` = 意图不固定时现生成一条**；`/kickoff` = 从零起新项目的完整流水线。

## 用法

```
/loop-gen <一句话意图>
```
例：
- `/loop-gen 把现有界面逐页还原成 prototype/ 的原型（截图比对修正）`
- `/loop-gen 把 cargo clippy 的 warning 一个个清干净`
- `/loop-gen 给 src/services 下每个模块补单测`
- `/loop-gen 把 XXX 模块从回调重构成 async/await`
- `/loop-gen 把 docs/ 里的旧术语 A 全量替换成 B 并逐文件核对`

## 生成步骤

1. **归类意图**到一种循环范式：建功能 / 还原 UI / 修 bug / 补测试 / 重构 / 迁移 / 清理 / 文档。
2. **摸项目**（真相源、验证命令、技术栈），缺关键信息就**只问你 1-2 个**：
   - 每轮「一项」的粒度？（一页 / 一个文件 / 一个 warning / 一个模块）
   - 每轮「算完成」的判据？（编译过 / 测试过 / 截图对齐原型 / warning 归零 / diff 干净）
   - 范围？（全量 / 某目录 / 某清单）
3. **套通用骨架**（见下），按意图把 `<尖括号>` 占位填实。
4. **UI 类意图自动加「原型保真截图闭环」**；非 UI 不加（别画蛇添足）。
5. **整段输出 `/loop` 提示词**给用户，并说明：在本项目新开会话粘贴即可；想圈范围也可在末尾自己加。

## 通用骨架（生成时按意图填空，不要原样吐占位符）

```
你是「<项目名>」的实现工程师。这是一个 /loop 循环：每触发一轮只做**一个最小可验证单元**，做完提交、结束本轮，严禁一轮多做。

== 真相源（每轮开工必读，不可凭记忆）==
- <按意图填：需求文档 / 原型 prototype/* / 待修清单 / 待重构模块列表 / 旧术语清单 …>
- 项目规约：CLAUDE.md（+ .claude/PROJECT.md / AGENTS.md，若有）

== 每轮标准流程 ==
1. 选下一个未完成的「<单元：页 / 文件 / warning / 模块 / 出现点>」（有清单按清单顺序；没有就扫描定位）。
2. 只做这一个最小单元，遵守项目技术栈与架构规约。
3. 验证门（必须过才提交）：<按意图填：tsc/cargo check/cmake --build 编译过 ‖ 对应测试通过 ‖ 截图与原型对齐 ‖ 该 warning 消失 ‖ 该处 diff 符合预期>。
4. 记录进度：<勾任务台账 / 在清单上划掉这一项 / BUILD-LOG 记一行>。
5. 最小提交：逐个 git add 本轮改动的具体文件（禁止 git add -A/.），commit（带 scope，一句话）。
6. 输出本轮小结：✅做了啥 | 📊进度 X/总 | ⏭️下一个。

<若意图涉及 UI/还原原型，追加这一段；否则删掉：
== 原型保真截图闭环（UI 任务必过）==
- 起前端 dev server（run_in_background，不 kill 已有进程）；
- web 前端：用 aicoder 内置浏览器 mcp__aicoder__browser_navigate 打开运行页 → mcp__aicoder__browser_screenshot 截「现状」；原生窗口（Qt）：用桌面截图 / QWidget::grab 截运行窗口；
- 打开对应原型当「目标」，逐条比对：布局分区/主辅状态色/间距留白/字号字重行高/控件类型/文案/交互态；
- 有差异只改视图层 → 重新截图确认消除；关键差异没消除就继续修、别急着提交。>

== 决策分流 ==
- 可逆/低风险 → 按合理默认继续，记一行 docs/BLOCKERS.md，不空等。
- 不可逆/合规/架构级/缺外部凭据 → 停下问我。

== 卡点熔断 ==
- 连续 3 轮无实质进展，或同一项 2 轮验证门修不过 → 停止循环，报错误原文 + 判断 + 建议。

== 停止条件 ==
- <按意图填完成判据：全部页与原型对齐 ‖ clippy warning 归零 ‖ 每个模块都有测试且通过 ‖ 清单全部处理完 …> 全满足 → 输出「✅ 完成」+ 自检，停止循环（不再续期）。

现在开始：<按意图给出第一步的具体动作，如"先列出待还原的页面清单/待修 warning 清单，再从第一项开始">。
```

## 约束

- 生成的提示词必须**填实**所有 `<尖括号>` 占位（用项目真实路径 / 命令），不要把占位符原样给用户。
- 验证命令要贴本项目技术栈（Tauri/web：`cargo check` + `tsc`；Qt：`cmake --build` + `ctest`）。
- 反捏造：不确定的真相源路径或完成判据，先问用户，别瞎填。
- 本命令只**生成提示词**，不替用户起飞循环（生成后由用户粘贴运行）。
